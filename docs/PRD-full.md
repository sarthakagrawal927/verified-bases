## 2. One-Line Description

A curated marketplace where users can preview, use, buy, own, remix, and launch verified software Bases built by creators.

## 3. Core Thesis

AI will make software generation cheap, fast, and abundant.

That does not eliminate value. It moves value away from raw code generation and toward:

* creator judgment
* verified outcomes
* trusted packaging
* live previews
* source ownership
* remixability
* launch support
* curation
* honest alternatives

The marketplace should not sell "AI-generated code." That will become a commodity.

It should sell verified, scoped, ownable software outcomes.

Internal thesis:

> Creator judgment + verified outcome + fading creator time + fading creator tokens.

Customer-facing thesis:

> Skip the blank prompt. Start from verified software.

## 5. What Is a Base?

A Base is a verified working software starting point.

It can be:

* website
* landing page
* internal tool
* Mac app
* mobile prototype
* script
* dashboard
* local-first app
* small business workflow
* creator/operator tool

A Base is not:

* a generic template
* a full SaaS replacement
* a random AI-generated repo
* a clone of a paid product
* an unsupported ZIP file
* a promise that everything can be customized perfectly

A Base includes some combination of:

* live preview
* demo video
* screenshots
* verified working build
* source-code tier
* setup guide
* known limitations
* alternatives
* DIY-with-AI estimate
* remix paths
* launch support options

## 6. Core User Problem

AI tools make it easier to build software from scratch, but users still face uncertainty.

They ask:

* Will the AI actually build the right thing?
* How long will it take?
* Will it deploy?
* Will the code be usable?
* What edge cases will break?
* Is this better than a free tool?
* Should I use SaaS instead?
* Can I customize it later?
* Can I own the source code?
* Can someone help me launch it?

The product solves this by giving them a verified working starting point instead of a blank prompt.

## 7. Core Customer Promise

You can probably build something similar yourself.

This exists for when you want:

* a working preview
* a clear scope
* source-code ownership
* honest alternatives
* known limitations

You are not paying because you cannot build it.

You are paying because someone already got the first useful version working.

## 10. Product Principles

### 1. Verified over abundant

Do not maximize listing count. Maximize trust.

### 2. Specific over generic

Reject generic "todo app," "CRM," "chatbot," and "booking app" unless they have a strong niche or ownership angle.

### 3. Honest alternatives

Every serious listing should show free tools, SaaS alternatives, and whether the buyer should just prompt it themselves.

### 4. Ownership included

Source-code ownership should be central to the brand.

### 5. Hosting is optional

Hosting should be framed as launch convenience, not the main product.

### 6. Remixing is scoped

Do not promise "customize anything." Define supported remix paths.

### 7. No clone-store positioning

Use existing SaaS as demand maps, not as products to copy.

### 8. Support must be bounded

Non-technical users need support, but support must be productized and limited.

## 11. Marketplace Differentiation

### Versus Lovable / Claude / Codex

They help users create from a prompt.

This marketplace starts from a verified result.

Positioning:

> AI builders start with a prompt. We start with a working Base.

> Claude can generate the app. We help you find the version worth owning.

> Lovable gives you the blank canvas. Bases give you the finished first pass.

### Versus Gumroad

Gumroad sells files and digital products broadly.

This marketplace sells verified working software packages with previews, source-code tiers, remix paths, and launch support.

### Versus CodeCanyon / old code marketplaces

Old marketplaces sell static code/scripts.

This marketplace sells live, verified, previewable, remixable, ownable software Bases.

### Versus SaaS

SaaS gives convenience and support, but usually no ownership.

This gives ownership, source access, customization, and optional launch help.

### Versus free big-tech tools

Free tools are better for generic use cases.

Bases are for smaller, specific, ownable, privacy-friendly, or customizable workflows.

## 12. Core Product Loop

### Buyer Loop

1. User browses Bases.
2. User opens a listing.
3. User sees live preview, demo, screenshots, alternatives, and DIY-with-AI estimate.
4. User chooses one of the tiers:

   * Preview
   * Use It
   * Own It
   * Remix It
   * Launch It
5. User receives app/source/remix/launch support.
6. User reviews the Base or requests further help.

### Creator Loop

1. Creator submits a Base.
2. Platform verifies quality.
3. Base gets listed.
4. Buyers preview and purchase.
5. Creator earns from use/source/remix/support tiers.
6. Platform tracks purchases, refunds, reviews, and support load.
7. High-performing Bases get featured.

Price: $5–$49 depending on product

Purpose:

Examples:

* downloadable Mac app
* compiled binary
* static website files
* hosted demo access
* packaged script
* usable app build

Important:

This tier should not imply unlimited customization.

### Tier 3: Own It

Price: $29–$199+

Purpose:

* source-code ownership
* GitHub handoff
* commercial use

Includes:

* GitHub repo access
* setup guide
* environment variable guide
* deploy guide
* license
* known limitations
* suggested customization paths

Brand promise:

> Buy once. Own your copy. Modify it. Use it commercially.

### Tier 4: Remix It

Price: $10–$99 per remix run

Purpose:

* AI-assisted customization within supported scope

Examples:

* change branding
* change copy
* add/remove fields
* adapt to a niche
* replace payment provider
* add simple workflow
* change theme
* add export
* add email notification

Public label:

> Remix Run

Avoid calling it "attempt" in buyer-facing copy.

Expectation:

A Remix Run creates a customized preview or branch based on user instructions. Some changes may require additional work if outside the supported remix scope.

### Tier 5: Launch It

Price: $99–$499+

Purpose:

* help user deploy or launch the Base

Includes:

* guided setup
* deployment to user's account
* domain configuration
* env setup
* database setup
* payment/email provider setup
* short launch checklist

Important:

Prefer launching into user-owned infrastructure first.

### Tier 6: Managed Hosting

Price: $10–$99/month

Purpose:

* convenience for non-technical users

Only available for Hosted Eligible Bases.

Includes:

* hosting
* basic uptime
* backups
* domain setup
* minor maintenance
* limited support

Hard rule:

Do not host arbitrary stacks.

## 17. Anti-Clone Positioning

The marketplace should not be positioned as:

> Cheap clones of paid software.

It should be positioned as:

> Ownable alternatives to SaaS workflows.

Use paid software as demand signals, not as things to copy.

Bad:

> Calendly clone

Good:

> Solo clinic booking Base with intake forms, deposits, private notes, and CSV export.

Bad:

> Notion clone

Good:

> Private team wiki Base with markdown pages, search, and role-based access.

Bad:

> Typeform clone

Good:

> Embeddable feedback form Base with dashboard, email alerts, and CSV export.

## 30. Risks

### Risk 1: Users prompt it themselves

Mitigation:

* focus on verified outcomes
* show what AI usually gets wrong
* sell source/remix/launch help
* target buyers who want results, not tinkering

### Risk 2: AI generation gets too good

Mitigation:

* do not rely on time/token savings
* build trust layer
* build curation layer
* build launch/support layer
* focus on verified packaged outcomes

### Risk 3: Marketplace fills with AI slop

Mitigation:

* curated approval
* strict review
* reject generic apps
* mandatory known limitations
* verification badges
* no open listing dump

### Risk 4: Hosting creates support nightmare

Mitigation:

* host only approved stacks
* start with launch help
* separate support from hosting
* no arbitrary hosting
* clear support boundaries

### Risk 5: Legal/IP issues from clones

Mitigation:

* anti-clone policy
* reject copied UI/assets/code/branding
* allow workflow alternatives, not product copies
* require original implementation
* avoid logos unless legally cleared
* include affiliate/non-affiliation disclaimers when relevant

### Risk 6: Non-technical users expect full SaaS

Mitigation:

* clear limitations
* alternatives section
* support tiers
* launch help
* "not for you if" copy

### Risk 7: Supply is easy, demand is weak

Mitigation:

* start with 10–20 curated Bases
* test paid demand before building full marketplace
* measure preview-to-purchase and source/remix/launch conversion
