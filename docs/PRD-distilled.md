# PRD: Verified Software Bases

## Concept

A curated marketplace for working software Bases that users can preview, use, buy, own, remix, and launch.

A **Base** is a verified software starting point: not a full SaaS, not a generic template, and not a random AI-generated repo.

## Core Thesis

As AI makes software generation cheap, raw code becomes less valuable.

The value shifts to:

* creator judgment
* verified outcomes
* working previews
* source ownership
* remixability
* launch support
* honest comparisons
* curation

Internal thesis:

> Creator judgment + verified outcome + fading creator time + fading creator tokens.

Customer-facing thesis:

> Skip the blank prompt. Start from verified software.

## Positioning

### Hero

> Skip the blank prompt. Start from verified software.

### Subheadline

> Browse curated software Bases that already work. Preview them, use them, buy the source code, remix them with AI, or get help launching them.

### Simple Explanation

You can probably ask AI to build something similar.
This is for when you want the working version, the source code, the tradeoffs, and a path to launch.

## What Users Can Buy

### 1. Live Preview

Free preview, demo, screenshots, or TestFlight-style access.

### 2. Use It

A usable app, website, Mac app, build, or packaged tool.

### 3. Own It

Source code, GitHub access, setup guide, license, and deploy docs.

### 4. Remix It

Paid AI-assisted customization within supported scope.

### 5. Launch It

Paid setup/deployment help, preferably into the buyer's own accounts.

### 6. Managed Hosting

Later. Only for approved, standardized Bases.

## Differentiation

### Versus Claude/Codex/Lovable

They start with a prompt.
This starts with a verified result.

### Versus Gumroad

Gumroad sells files.
This sells working, previewable, source-ownable software Bases.

### Versus SaaS

SaaS is polished and supported, but rented.
Bases are smaller, ownable, remixable, and launchable.

### Versus Code Marketplaces

Old code marketplaces sell static code.
Bases sell verified software outcomes with previews, limitations, and launch paths.

## Review Standard

Do not allow an open dump of AI slop.

Accept:

* useful small apps
* local-first tools
* Mac apps
* internal tools
* website Bases
* mobile prototypes

* SaaS-inspired workflows

Reject:

* clones
* copied UI/assets/code
* generic todo/CRM/chatbot slop
* broken demos
* unclear licensing
* misleading comparisons
* unsupported junk

## Final Definition

Verified Software Bases is a curated marketplace for working software starting points.

Buyers can preview a Base, use it, own the source code, remix it with AI, or get help launching it.

The product does not depend on tokens staying expensive. It bets that even when code generation is cheap, people still pay for judgment, verification, packaging, ownership, and launch support.
